What It Is

A Qt Widgets desktop app for managing an inventory of parts stored in a local JSON file, including images and related files. It presents a main window with dialogs for creating, editing, importing, and batch-updating parts.

Who It's For

People who maintain a parts inventory and want a local, file-based catalog with searchable metadata and attachments.

What It Does

  • Create and edit parts with rich fields (name, category, supplier, storage, quantity, price, links).
  • Search and filter parts by text and category; optional start-with-random part view.
  • Store data in JSON with auto IDs and timestamps; choose and reveal the database path.
  • Attach a primary image, additional images, and local files; open attachments from the UI.
  • Import parts from CSV with column mapping, preview, and duplicate detection.
  • Batch-update selected parts (category, subcategory, format, type, storage).
  • Manage preset lists for combo boxes (categories, formats, suppliers, etc.).

How It Works (Architecture)

  • UI Layer: Qt Widgets MainWindow plus dialogs (New Part, Settings, Import, List Manager, Batch Change).
  • Data Layer: JsonPartRepository handles CRUD, search, and persistence to partorium.json.
  • Presets: separate JSON file partorium_lists.json for list values used in combo boxes.
  • Settings: QSettings stores UI preferences and default folders.
  • Data Flow: UI actions -> JsonPartRepository -> JSON files on disk -> UI refresh.

How To Run (Minimal)

  • Install Qt 6 or Qt 5 with Widgets and Concurrent modules (per CMake configuration).
  • Configure and build with CMake using src/CMakeLists.txt (exact commands not found in repo).
  • Run the Partorium executable produced by the build.
